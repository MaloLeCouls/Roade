# 05 — Audit heuristique

> Constats UX précis, sourcés à `fichier:ligne`, croisant les **10 heuristiques de Nielsen** (N1…N10) et les **principes P1–P13** du référentiel (cf. `ROADE_GUIDE_CLAUDE_CODE.md`). Chaque ligne décrit une situation **concrète** observable depuis le code. Diagnostic uniquement — pas de solution.

## Légende

- **Sévérité** : **B** bloquant (l'utilisateur perd des données, ne peut pas accomplir une tâche, ou subit un risque produit majeur) · **M** majeur (friction durable, contre-pro évident) · **m** mineur (gêne ponctuelle) · **c** cosmétique (sans impact fonctionnel).
- **Principes** : P1–P13 (référentiel Roade) ; N1–N10 (Nielsen).
- Les principes Nielsen utilisés : N1 visibilité, N2 monde réel, N3 contrôle/liberté, N4 cohérence, N5 prévention d'erreur, N6 reconnaissance plutôt que rappel, N7 flexibilité/efficacité, N8 esthétique minimale, N9 reconnaître/diagnostiquer/récupérer, N10 aide/documentation.

## Synthèse — où concentrer le travail (sans préempter l'étape 3)

Le **chemin heureux** est tenu à un niveau pro (atouts F1–F10). Les écarts se massent sur **trois familles** :

1. **Pertes silencieuses possibles** (constats #1, #4, #5, #9, #10, #11) — l'utilisateur ne sait pas qu'il *a perdu* quelque chose. C'est l'angle « confiance » de P12/P13, et c'est le plus dangereux pour un outil de données. À traiter d'abord.
2. **Pas d'aiguillage proactif** (constats #3, #8, #12) — pas d'undo, pas de stop, pas de validation pré-run : trois absences structurelles qui interdisent l'exploration sans peur (P5/P7). Conjuguées, elles font basculer l'expérience d'« éditeur » à « machine à un coup ».
3. **A11y + adressabilité** (constats #1, #2, #6, #15) — quatre signaux quasi-binaires d'un produit *amateur* qu'un évaluateur externe relèvera immédiatement (P1, P11).

Les frictions mineures (asymétries, raccourcis, découvrabilité) sont nombreuses mais résolvables individuellement par l'extraction d'un design system (06) et une couche de feedback unifiée — donc à grouper plus tard.
